# Asset Pipeline Guide

How assets are organized, where they come from, and how to add new ones.

## Directory Structure

```
assets/
  models/              3D models (.glb format)
    jasper.glb         Player character (Tripo, skeletal animation)
    mountain1.glb      Forest mountain (Tripo)
    mountain2.glb      Forest mountain variant (Tripo)
    large_rock.glb     Large rock decoration (Tripo)
    small_rock.glb     Small rock decoration (Tripo)
    yellow_flower.glb  Yellow flower decoration (Tripo)
    redbricks.glb      Subdivision tile model (Tripo)
    grass-block.glb    Forest tile model (Kenney)
    cement-platform.glb City/sublevel tile model (Kenney)
    tree_*.glb         Various tree models (Trellis/Tripo)
    city/              City-specific models (skyscrapers, taxi, cones, lights)
    suburban/           Subdivision houses and trees (Tripo)
    cave/              Cave sublevel models (Kenney)
    sewer/             Sewer/subway sublevel models (Kenney)

  configs/             Background parallax configuration (JSON)
    forest_bg.json     Mountains, trees, clouds, attenuation
    subdivision_bg.json Houses, trees, attenuation, sky overlay
    city_bg.json       Skyscrapers, buildings, attenuation, sky overlay

  levels/              Compiled level data
    compiled_levels.json   Tiles, enemies, stars, doors, props, lights

levels/
  jasperworld.ldtk     LDtk project file (visual level editor)

tools/
  ldtk_compiler/       Rust tool: compiles LDtk → compiled_levels.json
  backfill_ldtk.py     Python tool: populates LDtk from compiled_levels.json
```

## Model Sources

### Tripo AI Models
Custom 3D models generated using [Tripo](https://www.tripo3d.ai/). These have their own textures baked in and are typically center-anchored (origin at the model's midpoint).

Used for: player character, houses, rocks, flowers, mountains, taxi, cone, skyscrapers (a/b/c).

**Important:** Model generation costs credits. Don't suggest regenerating models casually.

### Trellis Models
Custom tree models generated with Trellis. Center-anchored like Tripo models.

Used for: tree_oak, tree_fat, tree_pine, tree_default.

### Kenney Asset Packs
Free CC0 models from [Kenney.nl](https://kenney.nl/). These are bottom-anchored (origin at the model's base, Y=0 at the bottom).

Used for: tile blocks, cave/sewer decorations, flowers, grass, bushes, mushrooms.

### Anchoring Matters
When placing models, you need to know if they're **center-anchored** or **bottom-anchored**:

- **Bottom-anchored** (Kenney): Place at the surface Y. The model extends upward.
- **Center-anchored** (Tripo/Trellis): Place at `surface_Y + (native_height * scale) / 2`. Otherwise the model sinks halfway into the ground.

## JSON Config Files

### Background Configs (`assets/configs/*.json`)

These control the parallax background layers. Each level has its own config file.

**Example: Adding a mountain to `forest_bg.json`**

Find the `"mountains"` array and add an entry:
```json
{"model": "models/mountain1.glb", "x": 500.0, "native_h": 0.723, "scale": 350.0}
```
- `model` — path to the GLB file (relative to `assets/`)
- `x` — world X position
- `native_h` — the model's native height (used for center-anchor offset)
- `scale` — uniform scale factor

**Example: Changing house count in `subdivision_bg.json`**

Find `"near_houses"` and change `"step"` to adjust spacing:
- Larger step = fewer houses, more spread out
- Smaller step = more houses, denser

No recompilation needed — just edit the JSON and restart the game.

### Compiled Levels (`assets/levels/compiled_levels.json`)

This file contains all gameplay entities. It's generated by the LDtk compiler but can also be edited by hand.

**Prop format:**
```json
{
  "model_id": "models/small_rock.glb",
  "x": -801.0, "y": -142.8, "z": -15.0,
  "scale_x": 5.0, "scale_y": 13.0, "scale_z": 13.0,
  "rotation_y": -1.5708,
  "foreground": true
}
```

**Key fields:**
- `x, y` — world position
- `z` — depth layer (-15 for ground props, 10 for foreground trees, 3 for sublevel props)
- `scale_x, scale_y, scale_z` — per-axis scale (accounts for rotation)
- `rotation_y` — rotation in radians (Tripo models use -1.5708 = -90 degrees to face the camera)
- `foreground` — if true, gets the ForegroundDecoration marker component

**Light format:**
```json
{
  "x": 5153.0, "y": 5144.0, "z": 3.0,
  "color": [1.0, 0.6, 0.2],
  "intensity": 200000.0
}
```

## How to Add New Content

### Adding a New Prop Model

1. Place the `.glb` file in `assets/models/` (or a subfolder)
2. Check its dimensions:
   ```bash
   python3 -c "
   import struct, json
   with open('assets/models/my_model.glb', 'rb') as f:
       f.read(12)
       cl = struct.unpack('<I', f.read(4))[0]; f.read(4)
       gltf = json.loads(f.read(cl))
       for acc in gltf.get('accessors', []):
           if acc.get('type') == 'VEC3' and 'min' in acc:
               mn, mx = acc['min'], acc['max']
               print(f'dims: X={mx[0]-mn[0]:.3f} Y={mx[1]-mn[1]:.3f} Z={mx[2]-mn[2]:.3f}')
               print(f'anchor: {\"center\" if mn[1] < -0.01 else \"bottom\"}')
               break
   "
   ```
3. Add a prop entry to `compiled_levels.json` in the appropriate level/layer
4. For Tripo models (center-anchored, front faces +X): set `rotation_y: -1.5708` and swap scale_x/scale_z

### Adding a Background Element

1. Place the model in `assets/models/`
2. Edit the appropriate `assets/configs/*_bg.json` file
3. Add the model to the relevant array (mountains, trees, houses, etc.)
4. Restart the game to see changes

### Using the LDtk Editor

1. Open `levels/jasperworld.ldtk` in [LDtk](https://ldtk.io/)
2. Place entities on the map (drag and drop)
3. Set field values by clicking on placed entities
4. Save in LDtk
5. Compile: `cargo run -p ldtk_compiler -- --input levels/jasperworld.ldtk --output assets/levels/compiled_levels.json`

### Tile Models

Tiles are the platforms Jasper walks on. Each level uses a specific tile model:

| Level | Model | Notes |
|-------|-------|-------|
| Forest | `grass-block.glb` | Green grass block (Kenney) |
| Subdivision | `redbricks.glb` | Red brick block (Tripo) |
| City | `cement-platform.glb` | Grey cement block (Kenney) |

Tile model assignment is in the Rust code (`level/mod.rs` → `tile_models_for_layer`), not in JSON.

## Coordinate System

- **World space**: X = horizontal, Y = vertical (up), Z = depth (toward camera)
- **Origin**: Each level has an origin point (e.g., Forest = -864, -200)
- **Tile size**: 18 world units per tile
- **Ground level**: `origin_y + num_ground_rows * 18` (typically 3 rows of ground)
- **LDtk space**: Y is flipped (top-down). The compiler handles the conversion.
